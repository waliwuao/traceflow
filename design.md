## 项目概述
- traceflow 是一个专为小团队设计的“多人协作进度管理系统”。它的核心理念是将“项目进度汇报”转化为“团队博客发布”。
- 每个用户可以创建项目组，也可以加入别人创建好了的项目组，项目组是可以嵌套的，但是最多只能嵌套两层，比如算法项目组下的视觉识别子项目组。用户可以自己编写博客，然后将博客关联到项目组上，作为进度汇报，也可以在别人的博客下评论。注意，如果一个博客关联了一个子项目组，那么它就自动关联了所有父项目组
- 超级用户可以删除评论、博客、账户或者修改信息
- 用户可以在网页在线编辑和渲染markdown,所有的博客都是markdown,不支持上传图片和视频，如有需要直接给出url,因为数据库资源紧张只希望储存文本信息
- 对于每一个用户，有一个像github一样绿色瓷砖记录，可以清晰的看到每一个用户在哪天上传了多少博客

## 技术链
- docker:为了方便部署，使用docker进行环境管理
- postgre:数据库使用postgre
- java-spring-boot+jpa+restful api后端开发
- React + Tailwind CSS + shadcn/ui前端开发
- github flow版本管理
